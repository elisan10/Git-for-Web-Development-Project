    1. What is Semantic HTML? 
        - Semantic HTML is the HTML markup to describe the content of the page.
    2. What is HTML used for? 
        - It is used to markup a document to be displayed on a webpage.
    3. What is an attribute and where do we put it? 
        - An attribute is a modifier that supplies data to an element. It is put in an opening tag.
    4. What is the h1 tag used for? How many times should I use it on a page?
        - An h1 tag is used to define the most important header. It should only be used once.
    5. Name two tags that have required attributes
        - <img> and <a>
    6. What do we put in the head of our HTML document? 
        - Usually we put the title of the document, the link between the document and another source, and the meta.
    7. What is an id? 
        - Id is an attribute that specifies a unique id for an element.
    8. What elements can I add an id to? 
        - Any HTML element. 
    9. How many times can I use the same id on a page? 
        - Only once.
    10. What is a class?
        - Class is an attribute that specifies classnames for an element.
    11. What elements can I add a class to? 
        - Any HTML element as well as elements in javascript.
    12. How many times can I use the same class on a page? 
        - Multiple times.
    13. How do I get my link to open in a new tab?
        - You add a target attribute that would define where to display the response.
    14. What is the alt attribute in the image tag used for?
        - To specify a text for the image.
    15. How do I reference an id?
        - With the # followed by the name of the id.
    16. What is the difference between a section and a div
        - Section describes the the content within whereas the div just contains.
    17. What is CSS used for? 
        - CSS is used to style an HTML document.
    18. How to we select an element? Example - every h2 on the page
        - You start with the HTML element name followed by open curly brackets. 
    19. What is the difference between a class and an id? - Give me an example of when I might use each one
        - The class you can use more than one time and the id you can only use once. If I want to apply the same value to two ordifferent elements, I can add a class. If I only want to asign a value to one element, that's when I would use id.
    20. How do we select classes in CSS?
        - You begin with a . followed by the class name you want to select.
    21. How do we select a p element with a single class of “human””?
        - p .human followed by open curly brackets.
    22. What is a parent child selector? When would this be useful? 
        - A selector that selects elements with a specific parent. When you want to make one element more important than the other.
    23. How do you select all links within a div with the class of sidebar?
        - .sidebar a {}
    24. What is a pseudo selector?
        - a keyword that is added to a selector that defines a special state of an element.
    25. What do we use the change the spacing between lines?
        - The line-height property.
    26. What do we use to change the spacing between letters?
        - The letter-spacing property.
    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
        - The text-transform property.
    28. How do I add a 1px border around my div that is dotted and black?
        - border: 1px dotted black;
    29. How do I select everything on the page? 
        - with the universal selector *
    30. How do I write a comment in CSS?
        - /* comment */
    31. How do I find out what file I am in, when I am using the command line? 
        - pwd
    32. Using the command line - how do I see a list of files/folders in my current folder?
        - ls
    33. How do I remove a file via the command line? Why do I have to be careful with this?
        -  rm filename. Be careful because once you remove it, it's gone forever. YOu can't get it back.
    34. Why should I use version control? 
        - To keep track of changes and makes sure you're working on the latest version.
    35. How often should I commit to github?
        - Whenever you're making big changes on the file. 
    36. What is the command we would use to push our repo up to github? 
        - git push -u origin filename
    37. Walk me through Lambda's git flow. 
        - You begin by forking the repository. Then you move into your command line and enter git clone followed by the http of the repository. 
            After you clone, you need to change directory into your newly created folder in order to branch out. You do this by entering checkout -b filename.
            Now you can work on your branch version. Once you are ready to commit, you have to get your changes ready before you commit. You do this by 
            typing in git add .. You're saying that you want to add all these changes. Then you enter git commit -m "message about changes". Now you're ready 
            to push those commits. You do this by entering git push -u origin filename. Once it is complete, you can go into your github where the forked version
            of the project is and create a pull request. You have to make sure that you are in your main not Lambda's before you pull.
